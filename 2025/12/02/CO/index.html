存储器的性能指标

  1. 访问时间（Access Time）：TA
  • 随机访问存储器：从给定地址到存储器完成读或写操作所需时间。
  • 其他类型：指将读写机构定位到目标位置所需的时间。
  1. 存储周期（Cycle Time）：TC
  • 仅对RAM而言，指两次访问存储单元间的最小时间间隔。
  • TC > TA
  1. 带宽（Bandwidth）／数据传输率（Transfer Rate）:
  • 一般的随机访问存储器：1／Cycle Time；
  • 其他类型：TN = TA
  • N/R， 带宽：N / TN
    TN：读写N Bits所需的平均时间
    TA：访问时间
    N：N Bits
    R：存储部件的数据传输率（ bits /s）

简单理解就是为了防止电容漏电导致数据缺失，每次读出数据时把读出数据放大再倒灌回去部分（Cs << Cd）

刷新方式：

  • 集中：将刷新周期分成两部分，一部分只访存，一部分只刷新

分布式与分散的区别在于每次CPU访存后并不一定要刷新

七、高速缓冲存储器

1. 局部性原理

**局部性原理 (Principle of Locality)**：一般来说，访存趋于聚集在一个较小的连续存储区域中。

  • 时间局部性： 刚被访问过的单元不久后可能再次被访问（如循环指令）。
  • 空间局部性： 刚被访问过的单元的邻近单元可能不久被访问（如数组遍历）。

Cache 产生的背景：主存慢 + 局部性原理 + 有高成本读写快的方法

2. Cache 的原理

基本原理： 在CPU和主存间设置一容量较小的高速缓存，其中总是存放最活跃（被频繁访问）的程序块和数据，大多数情况下，CPU能直接从这个高速缓存中取得指令和数据，而不必访问主存。

特点： 速度越快，成本越高，容量越小

  • Cache与主存之间以数据块（Block）为单位进行数据交换
  • 每个数据块包含若干个字或字节
  • Cache块大小和主存块大小相等

Cache要解决的问题：

  • 快速访问；
  • 与主存交换数据；
  • 以主存地址被访问，判断是否能提供CPU所需数据（若能则提供）；
  • 未命中能替换Cache中的内容。

Cache 核心术语

  • 数据块（Block）：Cache与主存交换数据的基本单位。由连续的若干字节组成，其大小通常取决于存储器总线宽度。

  • 标记（Tag）：存放主存地址的高位部分。用于在Cache查找时，校验当前行中的数据是否对应CPU请求的主存地址。

  • 有效位（Valid bit）：指示Cache行中的数据是否有效。若为0，表示该行未加载数据或数据已失效（例如缓存刷新）。

  • 脏位（Dirty bit）：仅用于写回（Write-back）策略。指示该行数据是否被CPU修改过。若置位，说明Cache数据较新而主存数据旧，该行被替换时必须先写回主存。

  • 行（Line）：Cache中的基本存储容器。每一行包含一个数据块（Block）以及对应的管理信息（如Tag、有效位、脏位等）。

  • 组（Set）：组相联映射中的概念。由若干个Cache行（Line）组成。主存块根据地址索引映射到特定的组，然后在该组内寻找空闲或匹配的行。

  • 路（Way）：一组（Set）内包含的Cache行数。例如“4路组相联”意味着一个组内有4个行，查找时可同时比较这4行的Tag。

  • 命中率（Hit rate）：CPU访问Cache时，成功在Cache中找到所需数据的次数占总访问次数的比例。

  • 缺失率（Miss rate）：CPU访问Cache时，未找到数据而需访问主存的次数占总访问次数的比例（即 ）。

4.替换机制

缺失损失： CPU访问Cache缺失时，必须等待数据装入Cache后才能访问Cache所花时间

取出块的时间： 第一个字的延迟时间（存储器访问）+ 块的剩余部分的传送时间

缺失处理（以读操作为例，写操作比较复杂）

  • 块装入后访问：缺失数据块中各字按顺序全部装入Cache后，再从Cache中访问所请求的字（也即引起缺失的字）
  • 尽早重启（early restart）：缺失数据块中各字按顺序装入 Cache，一旦所请求的字装入 Cache ，CPU立即访问该字，控制机构再继续传送剩余数据到 cache
  • 请求字优先（requested word first）：所请求的字先装入 Cache，CPU立即访问该字，控制机构再按照先从所请求字的下一个地址、再到块的起始地址的顺序继续传送剩余数据到 cache

替换块的选择范围

  • 直接映射： 映射对应位置
  • 组相联： 组内任一块
  • 全相联： 任选一块

替换策略

  • 最近最少使用法（LRU，Least-Recently Used）：记录每一个数据块的相对使用情况，最近没有被使用的块被替换。
  • 先进先出法（FIFO，First-In-First-Out）：最先装入数据的块被替换；
  • 最小使用频率法 （LFU，Least-Frequently Used）：记录每一个数据块
    的使用频率，使用次数最少的被替换。
  • 随机法（RAND，Random）：随机选择一个数据块进行替换

实现方法

LRU（计数器法）

  • Cache的每一块都设置一个计数器；
  • 被调入或被替换的块，其计数器清 0，而其它的计数器则加 1；
  • 访问命中时，所有块的计数值与命中块的计数值进行比较：
    （1）如果计数值小于命中块的计数值， 则该块的计数值加 1；
    （2）如果块的计数值大于命中块的计数值，则数值不变。
    最后将命中块的计数器清为0。
  • 需要替换时，则选择计数值最大的块来替换。

理解：只需要保证相对大小即可，所以为了防止无限增长做了区分

FIFO（计数器法）

  • 例如Solar－16/65机Cache采用组相联方式，每组4块，每块都设定一个两位的计数器
  • 当某块被装入或被替换时，该块的计数器清为0，而同组的其它各块的计数器均加1
  • 当需要替换时就选择计数值最大的块被替换掉

  • 写通过（写直达，Write Through）：写Cache的同时写主存，效率较低；
  • 写回（Write Back）：写操作只更新Cache，直到Block替换时才将整个Block写回主存，一般使用“脏位”（dirty bit）来表示Block在替换回主存之前是否被修改过。

块的大小与命中率关系

  • 一般而言，增加块大小将降低缺失率（因为空间局部性）
  • 但块大小达到一定程度时，缺失率会随块大小的继续增加而上升（因为Cache总容量限制，块数量会下降，导致块替换增加）；
  • 单纯增加块大小带来缺失代价（缺失损失）的增大。

八、虚拟存储系统

（一）辅助存储器（外存）

1. 概述

  • 特点：不直接与 CPU 交换信息；容量大、价格低、非易失性、速度慢。
  • 分类：
    • 磁表面存储器：硬盘、磁带（通过磁化方向记录 0/1）。
    • 光介质存储器：光盘、DVD。

2. 磁记录编码方式
即将二进制数据变换为磁头电流翻转规律的方式。

  • 主要方式：
    • **NRZ (不归零制)**：1/0 对应正/负电流。效率高，但无自同步能力。
    • **RZ (归零制)**：正/负脉冲。
    • PM (调相制) :利用跳变方向表示数据。
    • **FM (调频制)**：利用频率表示数据。

  • 编码效率：。
    • 效率高 单位长度存的位数多（密度大）。
    • NRZ 效率最高（100%），FM/PM 较低（50%）。
  • 自同步能力：能否从信号中提取时钟。
    • 能力强 抗干扰好，无需单独时钟线。
    • PM/FM 自同步能力强，NRZ 差。

（二）虚拟存储系统

1. 调度方式对比

方式 划分单位 地址结构 优缺点
页式 固定大小的页 (Page) 虚页号 + 页内地址 内存利用率高，管理简单；主要用于虚实地址交换
段式 逻辑结构的段 (Segment) 段号 + 段内地址 易于实现共享和保护，段长可变；容易产生碎片
段页式 先分段，段内再分页 段号 + 页号 + 页内 结合两者优点，但查表需多次，速度较慢

2. 页式虚拟存储器

  • 地址映射：
    • 逻辑地址 (虚地址,用户编程空间) = 虚页号 (VPN) + 页内偏移 (Offset)
    • 物理地址 (实地址,物理内存空间) = 实页号 (PPN) + 页内偏移 (Offset)
    • 注：页内偏移量在转换过程中保持不变。
  • **页表 (Page Table)**：
    • 存放在主存中。
    • 作用：记录 虚页号 实页号 的映射。
    • 表项内容：实页号、有效位 (Valid)、脏位 (Dirty/Modified)、访问位 (Used) 等。

3. 快表 (TLB)

  • 本质：页表的 Cache（SRAM构成，集成在CPU内）。
  • 目的：解决由“两次访存”（查页表+取数据）导致的速度慢问题。
  • 原理：利用程序局部性原理，存储最近使用的页表项。
  • 流程：
    1. CPU 给出虚页号。
    2. 先查 TLB：若命中，直接得到实页号（无须访问主存页表）。
    3. 未命中：去主存查页表，并把该表项填入 TLB（可能会替换旧项）。
  • TLB项内容：虚页号(Tag) + 实页号 + 控制位(有效/脏/引用)。
  • 关联方式：通常采用全相联或组相联映射（为了高命中率）。

九、总线与I/O系统

1. 总线 (Bus)

定义： 连接多个部件的信息传输线，是各部件共享的传输介质。

  • 分时发送： 某一时刻只允许一个部件发送信息。
  • 同时接收： 多个部件可同时从总线上接收信息。

分类：

  1. 片内总线： CPU内部寄存器与ALU之间的连接。
  2. 系统总线： 连接CPU、主存、I/O接口的公共通道。
    • 数据总线 (Data Bus)： 双向，传输数据，宽度决定数据传输率。
    • 地址总线 (Address Bus)： 单向，传输地址，宽度决定寻址范围。
    • 控制总线 (Control Bus)： 传输控制信号（读写、请求、响应、时钟等）。
  3. 通信总线： 用于计算机系统之间或与其他系统的通信。

性能指标：

  • 总线宽度： 数据总线的位数（如32位、64位）。
  • 标准传输率 (Bandwidth)： 每秒传输的最大字节量 (B/s)。
  • 同步方式： 同步（统一时钟）/ 异步（应答方式）。
  • 总线复用： 地址和数据共用传输线（分时复用）。

2. 总线控制与通信

总线传输的五个阶段：

  1. 请求总线： 发出申请。
  2. 总线仲裁： 决定使用权归属。
  3. 寻址： 发出地址和命令。
  4. 信息传送： 数据交换。
  5. 状态返回： 撤销信号，释放总线。

总线仲裁方式 (决定谁用总线)：

  • 链式查询方式：
    • 原理：BG（总线同意）信号串行向下传递。
    • 特点：线少、易扩充；但对故障敏感（由于断路），优先级固定（离总线控制器近的优先级高）。
  • 计数器定时查询方式：
    • 原理：计数器轮询设备地址。
    • 特点：优先级灵活（通过改变计数初值）；但线数较多。
  • 独立请求方式：
    • 原理：每个设备有独立的REQ和GNT线。
    • 特点：响应快、优先级灵活；但线数最多，控制复杂。

总线通信控制 (协调收发双方)：

  • 同步通信： 统一时钟，固定周期。适合速度相近的设备。
  • 异步通信： 无固定周期，采用请求/应答 (握手) 方式。分为不互锁、半互锁、全互锁。

3. I/O 接口 (I/O Interface)

功能：

  • 地址译码： 识别设备地址。
  • 数据缓冲： 匹配CPU和外设的速度差异。
  • 格式转换： 如串/并行转换、电平转换。
  • 控制与状态： 传递命令和状态信息。

I/O编址方式：

  1. 独立编址 (Isolated I/O)： I/O地址与存储器地址分开，需专用指令 (如 IN, OUT)。
  2. 统一编址 (Memory Mapped I/O)： I/O占用存储器地址空间，使用访存指令操作I/O，无需专用指令。

MIPS CPU 采用的是 统一编址

4. I/O 控制方式

【问题】 假设幼儿园一个老师带 9个孩子，要给每个孩子分3块水果糖，并且要孩子们把3块糖都吃完，那么她可以采用什么方法呢?

(1) 程序查询方式 (Polling)

  • 原理： CPU不断读取I/O接口的状态寄存器，直到设备就绪才进行数据传送。
  • 特点：
    • CPU与外设串行工作。
    • 踏步等待，CPU效率极低。
    • 硬件简单，无需中断机构。
  • 解释： 先给孩子甲一块糖，盯着甲吃完，然后再给第二块，等吃完第二块又给第三块。接着给孩子乙，其过程与孩子甲完全一样。

(2) 程序中断方式 (Interrupt)

  • 原理： 外设准备好后向CPU发中断请求，CPU暂停现行程序，转去执行中断服务程序，处理完后返回断点。
  • 特点：
    • CPU与外设并行工作（准备阶段）。
    • 支持多道程序和实时处理。
    • 中断响应时机： 当前指令执行结束。
  • 处理过程：
    1. 中断请求： 硬件触发器INTR。
    2. 中断响应 (硬件隐指令)： 关中断 保护PC (入栈) 向量地址送PC (转入ISR)。
    3. 中断服务 (软件)： 保护现场 (通用寄存器) 设备服务 恢复现场 开中断 返回 (RETI)。
  • 解释： 给每个孩子发一块糖各自去吃，并约定谁吃完后就向老师举手报告，再发第二块

(3) DMA方式 (直接存储器访问)

  • 原理： DMA控制器接管总线权，数据直接在内存和外设间传输，无需CPU干预（仅在开始和结束时参与）。
  • 特点：
    • 适合高速、批量的数据传输（如磁盘、显卡）。
    • 周期窃取 (Cycle Stealing)： DMA挪用一个总线周期传输一个字，不中断CPU程序运行，仅暂停访存。
  • DMA vs 中断：
    比较项 中断方式 DMA方式
    响应时机 指令执行结束后 存取周期结束后 (指令周期内均可)
    处理方式 软件 (程序指令) 硬件 (DMA控制器)
    现场保护 需要 (PC+寄存器) 不需要 (不破坏CPU状态)
    适用场景 处理异常、低速I/O 大批量数据传输
  • 解释： 给每个孩子拿3 块糖各自去吃，吃完3 块糖后再向老师报告。

(4) 通道方式 (Channel)

  • 原理： 通道是专用的I/O处理器，有独立的指令系统（通道程序），由OS管理。
  • 特点： CPU只需发出I/O指令，后续全由通道完成，进一步减轻CPU负担。
  • 分类：
    1. 选择通道： 独占式，连接高速设备（类似DMA，一次服务一台）。
    2. 字节多路通道： 时间片轮转，连接多台慢速设备（交替传送字节）。
    3. 数组多路通道： 结合上述两者，连接多台高速设备（成组交叉传送）。
  • 解释： 权力下放：把发糖的事交给另一个人分管，只是必要时老师才过问一下
